# STRUCTURED OUTPUT FROM AGENTS IN LANGCHAIN

By default, when you ask an LLM a question, it replies with free-form text:

"The latest AI news includes GPT-5's release and new research from DeepMind. Sources: openai.com, deepmind.com..."

That's fine for a human reader, but terrible for code. If you want to display the answer in one place and the sources in another, you'd have to parse that string with regex or string-splitting — fragile and error-prone.

The idea
A structured response says to the LLM: "Don't just talk to me. Fill in this form."

You define the "form" as a Python class (using Pydantic), and LangChain makes sure the model's output matches it exactly.

Documentation: https://docs.langchain.com/oss/python/langchain/structured-output

```python
from dotenv import load_dotenv
from langchain.agents import create_agent
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage
from langchain_tavily import TavilySearch

from typing import List

# Pydantic models describe the *shape* of data using Python classes.
# LangChain converts them to a JSON schema, hands that schema to the LLM,
# and then parses the model's JSON output back into real Python objects.
from pydantic import BaseModel, Field

load_dotenv()

# --- Structured response schema -------------------------------------------
# We want the agent's final answer to follow a predictable shape so our code
# can reliably read fields like `answer` and `sources` instead of parsing
# free-form text.
#
# We define that shape with Pydantic models below. The `description` on each
# field is sent to the LLM as part of the schema, so the model knows what
# each field means and how it should be filled in.

class Source(BaseModel):
    """One web source the agent used to support its answer."""
    url: str = Field(description="The URL of the source")


class AgentResponse(BaseModel):
    """The full structured answer the agent must return."""
    answer: str = Field(description="The answer generated by the agent")
    sources: List[Source] = Field(
        default_factory=list,
        description="List of sources used by the agent",
    )

llm = ChatOpenAI(model="gpt-3.5-turbo", temperature=0)

tools = [TavilySearch()]

# --- Build the agent -------------------------------------------------------
# `response_format=AgentResponse` is the key piece for structured output.
# It tells `create_agent` to:
#   1. Advertise the `AgentResponse` JSON schema to the model.
#   2. After the agent finishes its tool-calling loop, validate the final
#      output against that schema and parse it into an `AgentResponse`
#      instance.
#   3. Expose the parsed object on the result under the
#      `"structured_response"` key (see `main` below).
#
# Without `response_format`, the agent would just return free-form text in
# `result["messages"]` and you would have to parse it yourself.
agent = create_agent(model=llm, tools=tools, response_format=AgentResponse)


def main():
    result = agent.invoke(
        {"messages": HumanMessage(content="Tell me about the latest news in AI.")}
    )

    # Because we passed `response_format=AgentResponse`, the parsed Pydantic
    # object lives at `result["structured_response"]`. It is a real
    # `AgentResponse` instance, so we get attribute access and type safety
    # instead of digging through raw strings.
    print("Agent's Response:")
    print(result['structured_response'].answer)

    # `sources` is a `List[Source]`, so each `source` is a `Source` model
    # with a typed `.url` attribute.
    print("\nSources Used by the Agent:")
    for source in result['structured_response'].sources:
        print(f"- {source.url}")

if __name__ == "__main__":
    main()
```

Think of response_format as a contract between your code and the LLM:

"You can think, search the web, call tools, do whatever you need... but when you're done, your final answer must look like this."

That contract is what turns an unpredictable chatbot into a reliable building block for real applications.

We can verify that the response from the agent is now an `AgentResponse` type.
